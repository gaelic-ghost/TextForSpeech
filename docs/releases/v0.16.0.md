# TextForSpeech v0.16.0

## What changed

- added built-in scalar-type speech rules for common typed numeric tokens such as `f32`, `f64`, `i32`, `u32`, `isize`, and `usize`
- taught the shared identifier speech layer to expand math-heavy combinations such as `cosF32`, `sinF64`, and similar identifier fragments into more speakable phrases like `cosine float thirty two`
- changed line-only file references such as `MarvisTTSModel.swift:208` so the built-in narration now says `at line 208`, while preserving the existing `line ... column ...` phrasing for file references that include a column
- updated the README and normalization coverage to document and verify the new built-in behavior

## Breaking changes

- none

## Migration notes

- no code changes are required for package consumers
- callers that already rely on the built-in base profile will automatically pick up the new scalar, math-identifier, and line-reference pronunciations after upgrading
