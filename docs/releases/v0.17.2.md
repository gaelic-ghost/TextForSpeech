# TextForSpeech v0.17.2

## What changed

- preserved `cwd` and `repoRoot` shortening for markdown inline code spans that contain file-line references such as ``/repo/Sources/Foo.swift:12``
- narrowed embedded-code path routing so slash-operator snippets like ``a/b`` stay in the generic code-speech lane instead of being misclassified as file paths
- added focused regression coverage for inline file references and inline slash-operator markdown spans

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should keep treating embedded markdown file-path detection as a narrow path-specific lane rather than a blanket rule for any slash-containing token
