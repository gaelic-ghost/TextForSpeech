# TextForSpeech v0.18.8

## What changed

- added opt-in summary-aware normalization through the final `summarize` argument on the async text and source normalization entrypoints
- added `TextForSpeech.SummaryProvider` with `.foundationModels`, `.openAIResponses`, and `.codexExec` provider options
- added runtime summary-provider persistence with `runtime.summaryProvider.get()`, `runtime.summaryProvider.list()`, and `runtime.summaryProvider.set(_:)`
- renamed `TextForSpeech.Context` to `TextForSpeech.InputContext` so input-local path and format hints are distinct from `TextForSpeech.RequestContext`
- updated README, roadmap, maintainer notes, and tests to describe the single async normalization shape with `withContext`, `requestContext`, and `summarize: false` by default

## Breaking changes

- `TextForSpeech.Context` has been renamed to `TextForSpeech.InputContext`
- `TextForSpeech.Normalize.text(...)`, `TextForSpeech.Normalize.source(...)`, `runtime.normalize.text(...)`, and `runtime.normalize.source(...)` are async throwing entrypoints
- the public text normalization entrypoints no longer accept separate `format` or `nestedFormat` arguments; pass `TextForSpeech.InputContext(textFormat:nestedSourceFormat:)` through `withContext:` instead

## Migration notes

- replace `TextForSpeech.Context(...)` with `TextForSpeech.InputContext(...)`
- replace `context:` at public normalization call sites with `withContext:`
- add `try await` to normalization calls
- keep deterministic behavior by omitting `summarize` or passing `summarize: false`
- opt into summary-aware normalization by passing `summarize: true`; if no summary provider has been set on the runtime, the provider defaults to `.foundationModels`
