# TextForSpeech v0.18.9

## What changed

- refined the public API audit cleanup so built-in profile fragment internals stay out of the public symbol surface
- kept `summarize: Bool` as the explicit normalization request flag for opt-in summary-aware normalization
- renamed the summary backend enum to `TextForSpeech.SummarizationProvider`
- exposed the persisted provider setting through `runtime.summarizationProvider.get()`, `runtime.summarizationProvider.list()`, and `runtime.summarizationProvider.set(_:)`
- kept the default persisted summarization provider as `.foundationModels`
- recorded follow-up public API cleanup for `PersistedState` and simpler replacement authoring in `ROADMAP.md`

## Breaking changes

- `TextForSpeech.SummaryProvider` is now `TextForSpeech.SummarizationProvider`
- `runtime.summaryProvider` is now `runtime.summarizationProvider`
- public normalization provider overrides use `summarizationProvider:`; `summarize:` remains the boolean request flag

## Migration notes

- replace `TextForSpeech.SummaryProvider` references with `TextForSpeech.SummarizationProvider`
- replace `runtime.summaryProvider.get/list/set` with `runtime.summarizationProvider.get/list/set`
- keep existing `summarize: true` or `summarize: false` call sites as-is
- persisted state written with the legacy `summaryProvider` key still decodes and migrates to the current `summarizationProvider` setting
