# TextForSpeech v0.16.1

## What changed

- added built-in extension aliases for speech-hostile file suffixes including `.xcodeproj`, `.pbxproj`, `.xcworkspace`, `.xcconfig`, `.xcscheme`, `.xctestplan`, `.xcresult`, `.xcassets`, `.xcstrings`, `.xcprivacy`, `.dSYM`, `.mdx`, `.tsx`, `.jsx`, `.jsonc`, `.ipynb`, `.wasm`, `.sqlite`, and `.db`
- taught dotted-identifier detection to keep speaking rewritten hyphenated alias segments naturally after those extension aliases are applied
- split the built-in profile inventory into semantic-role fragment files under `Sources/TextForSpeech/Models/BuiltInProfiles/` and reduced the top-level built-in profile file to the public composition surface
- expanded model and normalization coverage around semantic-core composition, extension aliases, file-line references, and hyphenated dotted alias segments
- updated the README and maintainer notes to describe the new built-in extension behavior and the semantic-role split

## Breaking changes

- none

## Migration notes

- no code changes are required for package consumers
- callers that already rely on the built-in base profile will automatically pick up the new extension pronunciations after upgrading
- maintainers adding new built-in rules should now place them in the semantic-role fragment file that matches the rule kind instead of extending one large mixed built-in inventory
