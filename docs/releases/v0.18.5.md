# TextForSpeech v0.18.5

## What changed

- added generalized spoken currency handling for prefixed dollar, pound, and euro amounts, including decimal forms such as `$9.39`, `£3.72`, and `€2.05`
- added generalized spoken measured-value handling for numeric unit suffixes with optional single-space separation, including distance, weight, storage, and transfer-rate forms such as `42 km`, `83 mi`, `90 kg`, `256GB`, `8 MB`, and `64Gbps`
- taught the file-path detector not to steal `/s` bandwidth tokens such as `GB/s` or `Tb/s` before the measured-value normalizer can rewrite them
- expanded focused normalization coverage around plural-aware units, singular forms, transfer-rate aliases, and the new currency surface

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should treat currency amounts and measured values as semantic-core lexical policy alongside the existing scalar, path, URL, and identifier speech transforms
