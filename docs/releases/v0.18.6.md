# TextForSpeech v0.18.6

## What changed

- changed the public text and source normalization finish pass so it preserves explicit line breaks and paragraph breaks instead of flattening them into a single sentence stream
- kept inline whitespace cleanup in place so extra spacing still collapses safely within each line while commas, sentence-closing punctuation, and similar suffix punctuation stay attached to the spoken word they follow
- added focused regression coverage for preserved paragraph structure in both text and source normalization, alongside existing punctuation-boundary replacement coverage
- updated the maintainer notes to treat line-preserving whitespace cleanup as the package baseline

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should treat explicit newlines and blank lines as meaningful input structure that normalization must preserve unless a future API adds an opt-in flattening mode
