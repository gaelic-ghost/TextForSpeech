# TextForSpeech v0.18.2

## What changed

- taught path narration to speak relative prefixes naturally, so paths like `./Sources/Foo.swift` and `../README.md` become "current directory" and "parent directory" phrases instead of awkward dot-prefixed output
- preserved leading `./` and `../` markers during token-boundary trimming so relative paths keep flowing through the file-path and file-reference rules instead of degrading into generic identifier speech
- added focused regression coverage for relative-path narration, repeated relative-path compaction, inline relative file references, and whole-token replacements surrounded by punctuation

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should keep treating relative path prefixes as meaningful path context during token trimming and speech conversion, not as disposable punctuation
