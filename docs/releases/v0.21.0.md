# TextForSpeech v0.21.0

## What changed

- slimmed `TextForSpeech.RequestContext` to `source`, `topic`, `cwd`, `repoRoot`, and `attributes`
- removed the redundant `app`, `agent`, and `project` request-context fields
- added a general speech preface from `RequestContext.source` and `RequestContext.topic`
- kept `cwd` and `repoRoot` as silent path-shortening context instead of spoken source labels
- documented that Codex-specific hook payload cleanup is downstream-owned for now
- refreshed README, roadmap, and maintainer docs around the current request-context and normalization ownership model
- condensed completed roadmap milestones into the roadmap history section

## Breaking changes

- `TextForSpeech.RequestContext.app`, `agent`, and `project` have been removed.
- `TextForSpeech.Normalize.text(...)`, `TextForSpeech.Normalize.source(...)`, and the runtime normalization wrappers now add a short preface when `RequestContext.source` or `RequestContext.topic` is present.

## Migration notes

- Replace `RequestContext(app:)`, `RequestContext(agent:)`, and `RequestContext(project:)` call sites with `source`, `topic`, `repoRoot`, or `attributes` depending on the meaning the caller needs.
- Use `source` for the broad request origin, such as `codex` or `speak-swiftly-server`.
- Use `topic` for the human-facing subject that should be spoken in the preface.
- Use `repoRoot` and `cwd` only for path-aware normalization context.
- Keep Codex-hook-specific payload parsing in the hook script before text enters this package.
