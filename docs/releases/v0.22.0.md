# TextForSpeech v0.22.0

## What Changed

- added required `RequestContext.reqPurpose` metadata with `.speech`, `.audioFile`, and `.audioStream`
- added optional `RequestContext.prefacePolicy` with `.default`, `.always`, and `.never`
- made request-context prefaces purpose-aware: `.speech` and `.audioStream` include the `source`/`topic` preface by default, while `.audioFile` omits it by default

## Breaking Changes

- `TextForSpeech.RequestContext` initializers and decoded payloads now require `reqPurpose`.

## Migration Notes

- Use `.speech` for live audible playback requests.
- Use `.audioFile` for retained audio-file generation where provenance should stay metadata instead of spoken text.
- Use `.audioStream` for streaming audio requests that should behave like live speech by default.
- Use `prefacePolicy: .always` or `prefacePolicy: .never` only when a caller needs to override the purpose-based default.
