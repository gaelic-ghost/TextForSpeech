# TextForSpeech v0.18.4

- changed the shipped balanced CLI-flag narration so `-d` now speaks as `tack d` and `--version` now speaks as `double tack version`
- kept the compact style terse and the explicit style on `short flag` / `long flag` phrasing while updating focused coverage for the new balanced wording
- documented the new balanced CLI baseline in the README and maintainer notes

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should treat the spoken `tack` / `double tack` wording as the balanced-style baseline for CLI flag prefixes
