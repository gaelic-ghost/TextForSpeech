# TextForSpeech v0.13.0

## What changed

- added a request-scoped repeated-path compaction pass ahead of built-in path speaking in the mixed-text lane
- taught repeated exact path mentions to collapse to `same path` instead of repeating the full spoken path again
- taught sibling paths under a previously spoken directory anchor to collapse to `same directory, …` so long repeated prefixes are spoken once per utterance instead of once per path
- updated the README and end-to-end tests to describe and verify the new repeated-path behavior

## Breaking changes

- none

## Migration notes

- callers do not need API changes to benefit from repeated-path compaction; the behavior is now part of the default mixed-text normalization pipeline
- callers that rely on repeated full-path narration in mixed text should review their downstream expectations and golden outputs
