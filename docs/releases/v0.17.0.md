# TextForSpeech v0.17.0

## What changed

- removed the legacy public `TextForSpeech.Forensics` namespace and its `words(in:)` entrypoint now that `SpeakSwiftly` no longer depends on that compatibility surface
- deleted leftover helper code and tests that only existed to support the removed forensic API surface
- added checked-in SwiftFormat, SwiftLint, and Swift language version baseline files aligned with the current `SpeakSwiftly` repository tooling
- applied the new SwiftFormat baseline across the package and documented the formatter and linter workflow in the repository guides
- updated the roadmap and maintainer-facing guidance so `M6 Forensic surface cleanup` is closed out and the package surface is described consistently as `Normalize` plus `Runtime`

## Breaking changes

- removed `TextForSpeech.Forensics.words(in:)`

## Migration notes

- callers should stop importing or calling `TextForSpeech.Forensics` and rely on the package’s supported `Normalize` and `Runtime` surfaces instead
- `SpeakSwiftly` already moved off this compatibility surface before this release, so no additional integration work is required there
- maintainers should use the checked-in `.swiftformat`, `.swiftlint.yml`, and `.swift-version` files as the repository source of truth for style tooling

## Verification performed

- `swiftformat --lint --config .swiftformat .`
- `swiftlint lint --config .swiftlint.yml`
- `swift build`
- `swift test`
