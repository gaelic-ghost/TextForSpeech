# TextForSpeech v0.23.0

## What Changed

- hardened live summary providers with explicit untrusted-content prompt boundaries, input limits, output limits, and bounded `.codexExec` process execution
- added clearer `.foundationModels` availability checks before running `LanguageModelSession`
- documented that `.foundationModels` uses Apple's Foundation Models framework directly and does not use Writing Tools
- removed `RequestContext.RequestPurpose.audioStream`

## Breaking Changes

- `TextForSpeech.RequestContext.RequestPurpose.audioStream` has been removed. Use `.speech` for spoken output regardless of whether downstream audio is played locally, returned through an HTTP response stream, or delivered over LAN.
- Decoding `request_context.reqPurpose: "audioStream"` now fails. Callers should send `"speech"` and model output destination separately from TextForSpeech request-purpose metadata.

## Migration Notes

- Replace `.audioStream` with `.speech` at Swift call sites.
- Replace JSON payloads using `"reqPurpose": "audioStream"` with `"reqPurpose": "speech"`.
- Keep transport choices such as local playback, response streaming, and LAN streaming in the downstream service's own output-destination model.
- Downstream callers that enable `summarize: true` with live providers should redact sensitive text before calling TextForSpeech when needed.
