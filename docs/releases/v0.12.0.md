# TextForSpeech v0.12.0

## What changed

- added a selectable built-in `style` model with shipped `.balanced`, `.compact`, and `.explicit` presets
- split the built-in normalization layer into `semanticCore + style preset + active custom profile`
- persisted the selected built-in style alongside stored custom profiles in `TextForSpeech.Runtime`
- taught the shipped styles to differ on common coding-agent shapes such as function calls, issue references, file line references, and CLI flags
- updated the README and maintainer docs to describe the built-in style model and current shipped behavior

## Breaking changes

- direct normalization entry points now model shipped built-ins through `style:` plus optional `customProfile:` instead of treating the caller input as a pre-merged effective profile
- runtime-managed normalization now derives its effective profile from `builtInBase(style: builtInStyle) + active custom`, so callers should treat built-in style selection as a first-class runtime concern

## Migration notes

- callers doing direct normalization should prefer:
  - `TextForSpeech.Normalize.text(..., style: .balanced)`
  - `TextForSpeech.Normalize.source(..., style: .balanced)`
- callers layering custom overrides should pass the raw custom layer rather than a manually pre-merged base:
  - `customProfile: runtime.profiles.active()`
  - `style: runtime.profiles.builtInStyle`
- runtime callers can read and update the shipped built-in style through:
  - `runtime.profiles.builtInStyle`
  - `runtime.profiles.setBuiltInStyle(_:)`
