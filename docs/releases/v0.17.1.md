# TextForSpeech v0.17.1

## What changed

- restored file-path normalization so path separators collapse to spacing again instead of being spoken as `slash` or `backslash`
- repaired contextual path shortening so `current directory` and `repo root` prefixes stay speech-safe without reintroducing spoken separators
- fixed markdown inline and fenced code handling so backticked file paths and file references take the path-specific normalization lane instead of falling back to generic code speech
- documented the file-path speech invariant in the maintainer guidance and updated the README to match the corrected downstream-TTS behavior
- added regression coverage for plain-text paths, context-aware path shortening, and backticked path spans

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should treat spoken `slash` or `backslash` inside normalized file paths as a regression unless a future change explicitly requests that behavior
