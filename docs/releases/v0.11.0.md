# TextForSpeech v0.11.0

## What changed

- split the runtime implementation into smaller role-focused files for the core runtime type, profile operations, persistence operations, and storage helpers
- split normalization helpers into markdown passes, token passes, replacement-rule engine logic, and format-detection helpers
- made `TextForSpeech.Runtime.profiles` and `TextForSpeech.Runtime.persistence` public grouped entry points for external package consumers
- refreshed README, roadmap, and maintainer docs to match the current architecture and public API

## Breaking changes

- `TextForSpeech.Runtime.profiles` and `TextForSpeech.Runtime.persistence` are now the intended public entry points, and docs now treat them as such
- no additional source-level API break is intended in this release beyond the already-landed runtime and base-profile model from the previous profile-persistence work

## Migration notes

- callers using `TextForSpeech.Runtime` should prefer the grouped handles:
  - `runtime.profiles`
  - `runtime.persistence`
- callers doing direct normalization without a runtime can continue using:
  - `TextForSpeech.Normalize.text(...)`
  - `TextForSpeech.Normalize.source(...)`
- callers that want runtime-managed normalization should continue passing `runtime.profiles.effective()`
