# TextForSpeech v0.18.3

## What changed

- taught list normalization to expand priority markers such as `[P1]` into spoken phrases like `Priority Level One` when they appear at the start of markdown or plaintext list items
- covered bullet lists, numbered lists, plain one-item-per-line priority lists, and markdown task-list variants such as `- [ ] [P1]`
- kept this behavior in the structural normalizer instead of the built-in lexical profile layer, and documented that ownership boundary for maintainers

## Breaking changes

- none

## Migration notes

- no caller-side API changes are required for this release
- maintainers should keep list-prefix parsing and other document-structure rules in `Normalization/`, even when the spoken output they produce looks like a replacement
